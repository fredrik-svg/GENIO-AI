Installationsguide för n8n-flöde

Den här guiden hjälper dig att ansluta GENIO-AI till ett n8n-flöde via REST/webhook. Varje steg kontrolleras direkt från Raspberry Pi så att du kan verifiera att integrationen fungerar innan den används i produktion.

Översikt över kommunikationen

Appen skickar händelser till n8n via en webhook. När chatten används skickas både användarens meddelande och assistentens svar till webhooken. Om du anger en API-nyckel kan appen även göra autentiserade statuskontroller mot n8n:s REST-API.

  1. Bas-URL (t.ex. http://192.168.1.50:5678) används för alla tester.
  2. Webhook-sökväg (t.ex. /webhook/genio-ai) tar emot händelser från assistenten.
  3. API-nyckel (valfri men rekommenderad) möjliggör kontrollfrågor mot /rest/workflows.
  4. När samtliga steg är godkända sparas konfigurationen i n8n_config.yaml och aktiveras direkt.

Steg 1 – Testa anslutning till n8n

Ange bas-URL till din n8n-instans. Guiden försöker sedan nå /healthz.

Steg 2 – Testa webhook

När bas-URL fungerar kan du ange webhook-sökvägen som flödet lyssnar på. Ett testpayload skickas för att bekräfta att n8n tar emot anropet.

Steg 3 – Verifiera API-nyckel (valfritt)

Om du använder n8n:s API-nyckel anger du den här för att testköra ett anrop mot /rest/workflows. Steget kan hoppas över om flödet är öppet för webhookar utan extra autentisering.

Steg 4 – Spara och aktivera

När du är nöjd sparas värdena i en lokal konfigurationsfil. Webhooken aktiveras direkt och kommande chattmeddelanden skickas till flödet.
